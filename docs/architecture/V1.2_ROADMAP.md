# Yo Te Invito — Backend Roadmap V1.2

## Objective

Backend V1.2 focuses on **hardening the ticketing platform and completing the backend MVP** before integrating real payment providers.

The goal is to ensure the system is:

- operational for real events
- safe against common operational errors
- observable
- consistent with the architecture already implemented

Payment integrations (MercadoPago, GetNet, etc.) will be implemented in a later phase.

# Slice 14 — Ticket Revocation + Refund Safety

### Objective
Allow administrators or producers to revoke tickets when necessary.

### Features
- New ticket status: `REVOKED`
- Ticket revocation endpoint
- Scanner returns `revoked` if scanned

### API


POST /admin/tickets/:ticketId/revoke


### Rules

- USED tickets cannot be revoked
- REVOKED tickets cannot be scanned
- Revocation must be logged in AuditLog

---

# Slice 15 — Event Capacity Guard

### Objective
Prevent events from exceeding the venue capacity.

### Changes

Add to Event:


capacityTotal


### Validation

Total attendees must satisfy:


ticketsSold + courtesies <= capacityTotal


---

# Slice 16 — Event Sales Metrics

### Objective
Provide producers with basic event metrics.

### API


GET /producer/events/:eventId/metrics


### Response


ticketsSold

# Slice 17 — Order Expiration Worker

### Problem
Orders currently expire only when payment is attempted.

This can lock ticket inventory.

### Solution

Background worker that runs every few minutes:


status = PENDING_PAYMENT
expiresAt < now


### Action


order.status = EXPIRED
release reserved stock


---

# Slice 18 — Ticket Transfer (Optional but Valuable)

### Objective
Allow users to transfer a ticket to another person.

### API


POST /tickets/:ticketId/transfer


### Rules

Ticket must be:


VALID
NOT USED
NOT REVOKED

# Slice 19 — Fraud Protection (Basic)

### Objective
Detect suspicious scanning patterns.

### Changes

Extend `TicketScanLog` with:

userAgent


### Detection examples

- excessive scans per second
- suspicious repeated scans
- invalid scan bursts
